# 手动发布 Netlify 部署指南

## 问题说明

自动部署脚本已经成功上传文件到 Netlify，但部署可能显示为 DRAFT（草稿）状态，需要手动发布。

## 快速解决方案

### 方法 1：在 Netlify Dashboard 手动发布（推荐）

1. **访问 Netlify Dashboard**
   - 打开：https://app.netlify.com
   - 登录你的账户

2. **选择你的站点**
   - 找到站点：propkitai.tech
   - 点击进入

3. **查看部署列表**
   - 点击 "Deploys" 标签
   - 查看最新的部署（应该在顶部）

4. **发布部署**
   - 如果部署显示为 "Draft" 或 "Ready"
   - 点击部署右侧的 "..." 菜单
   - 选择 "Publish deploy"
   - 或直接点击 "Publish deploy" 按钮

5. **等待生效**
   - 等待 1-2 分钟
   - 清除浏览器缓存（Ctrl+F5）
   - 或使用无痕窗口访问

### 方法 2：使用 Netlify CLI（如果已安装）

```cmd
netlify deploy --prod
```

## 验证部署

部署发布后：

1. **等待 1-2 分钟**（CDN 传播时间）

2. **清除浏览器缓存**
   - Windows: Ctrl + Shift + Delete
   - 或硬刷新: Ctrl + F5

3. **使用无痕窗口**
   - 打开新的无痕/隐私窗口
   - 访问：https://propkitai.tech

4. **验证内容**
   - 应该看到 "PropKit Analytics" 主页
   - 显示三个平台卡片（Alpha, AI, API）
   - 不再是 "Listing Magic AI"

## 如果还是显示旧内容

1. **检查部署状态**
   - 运行：`C:\ProgramData\Anaconda3\python.exe check_deploy_status.py`
   - 查看最新部署是否已发布

2. **检查 Netlify 设置**
   - 确保站点设置正确
   - 检查是否有构建钩子冲突

3. **尝试强制清除缓存**
   - 在 Netlify Dashboard 中
   - Site settings → Build & deploy
   - 点击 "Clear cache and deploy site"

## 自动化发布（未来改进）

目前自动发布功能可能因为 Netlify API 的限制而无法完全自动化。建议：

1. **使用 Git 自动部署**（最佳方案）
   - 连接 GitHub/GitLab
   - 每次 push 自动部署和发布

2. **手动发布**（当前方案）
   - 运行部署脚本上传文件
   - 在 Dashboard 手动发布
